# 按选定素材产出选题

这是一个选题提炼工作流：基于用户已选中或已采纳的笔记素材，为用户产出精准的选题卡。**本技能只负责产出"选题角度",不写标题、不写正文(那是 `xhs-copywriting`),也不自行落库(落库归主控 §4)。**

## 选题方法论(产出任何选题都遵循)

### 什么是"选题角度"(先分清，别串味)
- 选题角度 = **一句话说清"这篇讲什么方向、从哪个切口切入"**,是描述性的方向,不是成品标题。
  - ✅ 角度:"露营新手最容易忽略的 5 个装备坑"
  - ❌ 成品标题:"血泪!这 5 个露营坑我替你踩了"(≤20字、带钩子——那是写文案阶段的事)
- 选题卡里放**角度**,不放成品标题。

### 好选题的判定(每条都要过)
1. **有具体切口**:能指向一个明确人群/场景/痛点,不是"健身好处"这种泛主题。
2. **有认知落差**:让目标读者"咦?和我想的不一样"或"这正是我没搞懂的"。
3. **可被笔记验证**:这个角度在选中素材/库内能找到支撑(互动数据、真实案例),不是凭空想。
4. **可执行**:用户看完知道这篇大概怎么写、给谁看。

### 从爆款拆出可复用角度(拆解颗粒度)
对每篇高互动素材,拆这四样(单篇/多篇都用):
- **切入角度**:它从哪个口子切进去的(避坑/逆袭/对比/科普/清单…)。
- **戳中的痛点**:它替谁说出了什么没被满足的需求或焦虑。
- **标题/开头钩子的机制**:它靠什么勾住人(反常识/数字/身份代入/悬念…)——只拆机制,不照搬文字。
- **互动为什么成立**:高赞/高藏/高评分别说明什么(高藏=值得收藏的实用信息;高评=有争议或共鸣)。
把"机制"泛化成可复用的角度,而不是抄它的具体内容。

### 五类内容单元 → 选题装配
把素材里的信息归到五类,再组合成角度:
| 单元 | 是什么 | 适合的角度 |
|---|---|---|
| 观点 | 有立场的判断 | 争议/态度输出 |
| 案例 | 有人物/结果的真实故事 | 共鸣/逆袭/对比 |
| 方法 | 可操作步骤 | 教程/清单/攻略 |
| 概念 | 对某词/现象的定义 | 科普/认知升级 |
| 问题 | 值得深挖的疑问 | 好奇/悬念/互动 |
常用组合:观点+案例→"我怎么想+我怎么做到";方法+概念→"是什么+怎么做";问题+观点→"你以为+其实"。

### 差异化:一批 3~5 个怎么不重复
- **一个角度只占一个"切口×单元组合"**,批内不重样:换人群、换场景、换单元类型(别 5 条全是清单)。
- 用户说"再来一批/换角度/避免重复"时:**先回顾上一批已用过的切口和人群,新批必须换没用过的切口或单元组合**,不是把旧角度换个说法。
- 每条角度之间要能一眼看出区别;说不清区别的两条,合并或替换。

## 基于单篇原文出选题(搜索卡片「✨出选题」入口)

当用户从搜索卡片点「出选题」时,消息会带**一篇**笔记 of JSON(含 title/summary/author/互动/tags/note_url,本地笔记还带 resource_id)。此时走**单篇 → 单选题**快路:

1. 按上面《拆解颗粒度》分析这一篇(切入角度/痛点/钩子机制/互动成因)。
2. 产出 **1 个**可执行选题角度,按主控 §5 的 `xhs_topics` 结构输出(topics 数组仅 1 条),intro 一句话说明"基于〈这篇标题〉提炼"。
3. **依据(evidence)**:
   - 笔记带 `resource_id`(本地已收录)→ evidence 直接引用该 resource_id + 标题 + 摘要 + 时效。
   - 笔记只有 `note_url`(线上、未采纳)→ **不编 resource_id**;在 intro 或选题角度里注明"(线上实时:note_url)",并提示用户"这篇是线上实时的,采纳收录后我可以把它补成正式依据"。
4. **只展示,不落库**:不调 `save_generated_topic`/`sync_topic_to_feishu`。用户认可这个选题后(选定/说存档)才存,且只存这一个;若它基于线上笔记,提示先 `adopt_online_notes` 采纳那篇再补依据。

## 基于选定素材出选题

当用户在界面上勾选了多篇笔记，或显式要求“基于这几篇笔记出选题”时触发：

1. **用户已显式勾选指定的这一批 → 以这批为准,不要再换素材去做新的库内/全文检索稀释它**(可选地按下面《双源》拉线上趋势作启发,但主体是这批)。
2. 按《拆解颗粒度》逐篇拆,再按《五类内容单元→选题装配》组合,提炼爆款规律。
3. 产出 **3~5 个选题方向**,**批内按《差异化》规则互不重复**,按主控 §5 的 `xhs_topics` 代码块输出。
4. **只展示,不在此技能中落库**。

## 双源出选题:库内依据 + 线上趋势信号(贴热点用)

出选题时,正式依据**始终**来自库内 §6 证据检索(带 `resource_id`)。在此之外,**可选地**调 `search_xhs_online(方向)` 拉线上实时热门作为**趋势信号**——让选题贴当下热点:
- 线上趋势**瞬态、不落库、只作启发**,绝不作为正式 evidence。
- 纯靠线上趋势的角度,evidence 里只标 `note_url`、不编 `resource_id`,并提示用户"这是线上实时趋势,采纳收录后可补成正式依据"。
- 守两条铁律:**采纳才落库、正式依据须库内 resource_id**。
(注:用户已显式勾选某几篇时,趋势信号仅作轻参考,不喧宾夺主。)

## 数据不足时

检索口径与 mode 三态一律遵循主控 §6《检索与证据规约》。若库内没有足够相关素材(命中"数据不足"态),**明说"当前数据不足"**、建议先同步飞书资源或补充数据,**绝不编造来源/resource_id,也不拿弱相关硬凑选题**。

## 选题确认后的处理(职责边界:本技能只产出选题,落库归主控)

本技能的产出是**展示给用户挑选的选题卡**,默认**不落库、不同步飞书**。后续按两种场景处理,边界清晰、互不混淆:

### 场景一:用户要保存选题(发"保存该选题""存选题""存档",或点选题卡【保存】)
此时按主控《存储路由与权威性》(§4)落库——先写数据库、成功后同步飞书,飞书写经 HITL 人工确认:
1. `save_generated_topic`(只存用户选中的那个/那些,没选的丢弃;evidence 由前端经 InjectedState 直传,你无需也无法重填 resource_id)
2. `sync_topic_to_feishu`(同步飞书,经审批确认)
- 若选题基于**线上未采纳**笔记(只有 note_url、无 resource_id):先提示用户 `adopt_online_notes` 采纳,采纳后该笔记进检索,再作带 resource_id 的正式依据落库。

### 场景二:用户要写文案(发"写文案""写完整文案")
**本技能不执行任何落库动作**,直接把工作流交给后续文案创作流程,输出以下引导词:
```
已为您锁定这个选题。

下一步我可以继续帮您写完整正文：先给 3 个标题方向，再给开头方案，最后生成完整笔记。

您可以直接说：**【写完整文案】**。
```
(选题是否在转写前落库,由场景一决定;场景二本身只做流程衔接、不碰存储。)
