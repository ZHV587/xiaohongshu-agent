# 按方向产出选题 + 文案

这是一个两步式工作流:先给选题菜单,用户选定后再写完整文案。

## 工具边界与检索顺序

保持 DeepAgents 官方 skill/tool 边界:只使用智能体已暴露的工具,不直接访问 Postgres、
不调用 CLI,也不绕过工具边界读取数据。

1. 先调用 `search_resources` 检索统一 Postgres 数据底座,关键词结果是创作的基础。
2. 需要补充语义召回时可调用 `semantic_search_resources`;语义检索失败或不可用时,
   必须回退并继续使用已有的关键词结果。
3. 对选中的来源调用 `get_resource` 获取详情;仅当关联上下文能增加创作依据时调用
   `graph_expand`。
4. 仅当统一 Postgres 检索结果为空时,才可用 `read_xhs_data` 或 `read_feishu_wiki`
   直接读取飞书,并明确说明这些是回退数据。直接飞书读取只能作为回退,不能作为默认路径。
5. 如果统一检索和直接飞书回退均无可用来源,明确回复“当前数据不足”,建议调用
   `sync_feishu_resources` 同步飞书资源或补充数据,不得编造选题、文案或来源依据。

## 第一步:出选题(用户给方向后)

1. 按上述检索顺序查找并筛选来源,记录实际采用来源的资源 ID、标题、依据摘要和更新时间。来源未提供更新时间时写“未知”,不得猜测或伪造时间。
2. 用 `task` 委派 `baokuan-analyst` 子智能体基于这些来源拆解该方向爆款,
   要求把结论写到 `/analysis/<方向>.md`。
3. 用 `read_file` 读取该分析文件。
4. 基于分析,产出 **3~5 个选题方向**。**必须**用如下代码块输出(前端据此渲染成可点选卡片),
   intro 写一句引导语,topics 每项是"一句话角度（预期爆点）":

   ```xhs_topics
   {"intro": "根据你的爆款库，给你这几个方向，点一个我就展开写：", "topics": ["选题角度1（为什么可能火）", "选题角度2（……）"], "evidence": [{"resource_id": "资源ID", "title": "来源标题", "summary": "支持本次创作的关键依据", "updated_at": "2026-06-19T12:30:00+00:00"}]}
   ```

   代码块必须是合法 JSON、单独成段。
5. **停下,请用户选择一个选题。不要在这一步直接写完整文案。**

## 第二步:写文案(用户选定选题后)

为选定选题写完整小红书文案。**必须**用如下代码块输出(前端据此渲染成带一键复制的文案卡):

```xhs_copy
{"title": "小红书标题党风格，可带 emoji", "body": "分段、口语化、带 emoji、有记忆点的正文（用 \\n 表示换行）", "tags": ["#标签1", "#标签2"], "evidence": [{"resource_id": "资源ID", "title": "来源标题", "summary": "支持本次创作的关键依据", "updated_at": "2026-06-19T12:30:00+00:00"}]}
```

代码块必须是合法 JSON（正文里的换行用 \\n、引号转义）、单独成段。
`xhs_topics` 和 `xhs_copy` 的 `evidence` 数组只列实际采用的来源,每项固定包含
`resource_id`、`title`、`summary`、`updated_at`。

同时用 `write_file` 把这篇文案存到 `/drafts/<slug>.md`(slug 用方向+序号,如 `露营装备-1`)。

## 第三步:打磨

用户提意见时迭代修改当前文案,保持分块可复制格式。

## 风格沉淀(贯穿)

提炼到通用套路时,`read_file` 读 `/shared/xhs-style.md`。
不存在则 `write_file` 创建;已存在则用 `edit_file` 替换为追加后的完整内容
(StoreBackend 不允许对已存在文件重复 write_file),保留旧内容不丢弃。

## 质量检查(交付文案前)
- [ ] 标题有钩子,不平淡
- [ ] 正文像真人小红书笔记,无 AI 腔、无营销八股
- [ ] 标签 5~10 个且相关
- [ ] 选题与文案均有数据依据,非凭空
- [ ] 文案已存入 /drafts/
